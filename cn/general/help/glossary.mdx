## BeaconKit

BeaconKit 是模块化、可定制的共识层框架，利用 CometBFT 共识算法构建基于以太坊的区块链。

## BERA 代币

`$BERA` 是 Berachain L1 的原生 gas 代币，用途包括：

- 支付交易手续费
- 作为初始验证者质押代币以保障网络
  - 质押的 BERA 越多，提议的区块越多
  - 验证者每提议一个区块可获得基础排放与交易费（如 MEV）
- 可通过销毁 BGT 获得（单向转换）

更多见[代币 - $BERA](/cn/general/tokens/bera)。

## BEX

Berachain 的原生去中心化交易所。更多见[BEX 概述](/cn/bex/learn/overview)。

## BGT（Bera 治理代币）

`$BGT` 是 Berachain 的质押与治理代币，不可转让，只能通过参与流动性证明（PoL）获得：

- 验证者委托与奖励
  - 委托的 BGT 越多，奖励金库的奖励排放越多
- 参与治理（提案与投票）
- 可销毁换 `$BERA`（单向转换）
- 只能通过参与流动性证明获得

更多见[代币 - $BGT](/cn/general/tokens/bgt)。

## Block（区块）

按顺序永久写入区块链、包含交易列表的数据单元。

## Block time（出块时间）

在区块链上产生新区块所需的时间。Berachain 平均出块时间小于 3 秒。注意出块时间可能随网络拥堵增加。

## CometBFT

Berachain 使用的通用区块链共识引擎，用于实现高吞吐与快速最终性。更多见 [Cometbft.com](https://cometbft.com)。

## Consensus client（共识客户端）

共识客户端是负责在网络节点间就区块链当前状态达成一致的软件，负责验证交易与区块、确保其符合网络规则并决定哪些区块被加入链。

## Consensus mechanism（共识机制）

Berachain 网络节点就区块链状态达成一致的协议。Berachain 使用流动性证明，根据提供的流动性选择验证者。

## Delegation（委托）

代币持有者将投票或验证权授予网络中其他参与者的过程。

## DEX（去中心化交易所）

在区块链上直接买卖代币、无需中心化中介的平台。所有流动性由智能合约可验证地持有。

## Engine API

Engine API 是 EVM 节点执行层与共识层之间通信的接口。BeaconKit 作为共识层利用该接口便于与任意执行客户端配对。

## Execution client（执行客户端）

EVM（以太坊虚拟机）执行客户端是负责在区块内实际执行交易计算的软件，使用 EVM 解释并执行智能合约代码、管理状态变更并执行交易逻辑。

支持的 EVM 执行客户端：

- **Bera-Geth**：Berachain 对以太坊官方 Go 实现的 fork
- **Bera-Reth**：Berachain 对注重性能与可靠性的流行 Rust 客户端的 fork

## Finality（最终性）

交易在区块链上确认后不可更改或撤销的保证。Berachain 为交易提供即时最终性。

## Governance（治理）

Berachain 生态内做决策的体系。治理包括提案、投票以及使用 BGT 参与 PoL 与 Berachain 原生 dApp（BEX、HoneySwap）的变更实施。

## HONEY

`$HONEY` 是 Berachain 生态的原生稳定币，软锚定 1 USDC。在 Berachain 生态中广泛使用，涉及铸造与销毁费用。更多见[代币 - $HONEY](/cn/general/tokens/honey)。

## Liquidity（流动性）

Berachain 网络上可用于促进交易的流动资产的可获得性。流动性通常由用户通过流动性池提供。

## Liquidity pool（流动性池）

锁定在智能合约中的资金集合，用于在去中心化交易所及其他 DeFi 服务上促进交易。

## Liquidity provider（流动性提供者）

将代币存入流动性池的用户，从池内 swap 产生的手续费中获得一部分，以及可能来自 PoL 或其他来源的奖励。

## Mainnet（主网）

Berachain 区块链上发生真实价值交易的主要网络，与用于开发的测试网相对。

## Proof-of-Liquidity（流动性证明）

一种共识机制，使验证者、应用与用户的经济激励一致。基于双代币模型，验证者被选中的概率与其质押的 `$BERA` 数量相关。区块奖励以 `$BGT` 形式发放，数量受用户委托给验证者的 `$BGT` 影响。

## Single slot finality（单槽最终性）

在同一提议区块内达成最终性的过程。有时也称为*即时最终性*。

## Staking（质押）

锁定代币以支持区块链网络运行的过程。在 Berachain 中，质押用于保障网络并参与治理。

## Swap（兑换）

在去中心化交易所将一种代币兑换成另一种的过程。Swap 会产生费用，具体取决于池的设置。

## WBera staking（WBERA 质押）

通过存入 WBERA 质押金库，BERA 持有者获得直接收益机会：在激励分配过程中，协议激励的 33% 会自动重定向并分发给这些质押者。
